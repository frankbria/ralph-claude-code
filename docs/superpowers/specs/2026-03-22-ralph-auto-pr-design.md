# Ralph Auto-PR at End of Every Run

**Date:** 2026-03-22
**Status:** Approved
**Scope:** All Ralph variants (Claude, Codex, Devin) â€” interactive and loop modes

---

## Problem

Ralph currently ends each loop iteration by merging the worktree branch directly into the main branch after an interactive "Merge into main? (yes/no)" prompt. There is no audit trail, no code review gate, and no PR for the work done in each run. The desired end state is always a Pull Request â€” never a direct merge to main.

---

## Goal

At the end of every Ralph run (every loop iteration), Ralph must:
1. Commit all work in its working worktree branch
2. Push the branch to remote
3. Open a GitHub Pull Request targeting the main branch

This is **default behaviour** across all execution modes and all variants.

---

## Architecture

### New file: `lib/pr_manager.sh`

A shared library sourced by all three loop scripts. Contains all PR-related logic in one place.

**Functions:**

```
pr_preflight_check()
```
- Called once at loop startup
- Checks: `gh` CLI is installed, git remote exists, `gh auth status` passes
- On any failure: logs a loud warning block (uppercase, bordered), sets `PR_CAPABLE=false`
- Always returns 0 â€” never blocks execution
- Sets global `PR_CAPABLE=true|false` used by all subsequent PR calls

```
pr_build_title(task_id, task_name)
```
- Returns: `ralph: <task_name> [<task_id>]`
- Falls back to branch name if `fix_plan.md` yields no match

```
pr_build_description(branch, gate_results, gate_passed)
```
- Produces a structured Markdown PR body:
  - Task summary (task ID + name)
  - Quality gate results table: one row per gate with PASS âś… / FAIL âťŚ + exit code
  - If `gate_passed=false`: prominent failure section listing each failed gate command and exit code
  - Auto-generated footer: `đź¤– Generated by Ralph [engine] loop #<N>`
- `gate_results` is the semicolon-separated output already written to `.ralph/.quality_gate_results`

```
worktree_commit_and_pr(task_id, task_name, gate_passed, gate_results)
```
- Step 1: `git add -A && git commit` inside the worktree â€” skips if nothing to commit
- Step 2: `git push origin <branch> --set-upstream`
- Step 3: `gh pr create --base <PR_BASE_BRANCH> --title <title> --body <description> [--draft]`
- Step 4: if `gate_passed=false` â†’ `gh pr edit --add-label "quality-gates-failed"` (graceful if label doesn't exist)
- Logs PR URL on success
- If `PR_CAPABLE=false`: logs warning, commits and pushes only, skips `gh` calls

```
worktree_fallback_branch_pr(task_id, task_name)
```
- Used when `WORKTREE_ENABLED=false` (Ralph working directly on the current branch)
- Creates branch `ralph-<engine>/<task_id>-<timestamp>` off current HEAD
- Stashes any uncommitted changes â†’ checks out new branch â†’ pops stash â†’ commits â†’ pushes â†’ creates PR
- Handles the case where stash is empty (nothing to stash)

## Changes to Loop Scripts

Applies identically to all three variants:
- `ralph_loop.sh` (Claude)
- `codex/ralph_loop_codex.sh` (Codex)
- `devin/ralph_loop_devin.sh` (Devin)

### 1. Source + preflight at startup

```bash
source "$LIB_DIR/pr_manager.sh"
# ...existing sources...
pr_preflight_check   # called once before the main loop begins
```

### 2. Replace the merge prompt block

**Removed (current behaviour):**
```
quality gates pass  â†’ "Merge into main? (yes/no)" â†’ worktree_merge() â†’ worktree_cleanup(true)
quality gates fail  â†’ preserve branch â†’ worktree_cleanup(false)
```

**Replaced with (new behaviour):**
```
quality gates pass         â†’ worktree_commit_and_pr(task_id, task_name, true,  gate_results) â†’ worktree_cleanup(false)
quality gates fail (give up) â†’ worktree_commit_and_pr(task_id, task_name, false, gate_results) â†’ worktree_cleanup(false)
quality gates still resolving â†’ continue looping (no PR yet)
```

`worktree_cleanup(false)` is used in both cases â€” the branch is **always preserved** as the PR head. It is never deleted by Ralph.

### 3. Non-worktree fallback

At the end of a successful run when `WORKTREE_ENABLED=false`:
```bash
worktree_fallback_branch_pr "$task_id" "$task_name"
```

## Configuration (`lib/enable_core.sh` template â†’ `.ralphrc`)

Three new knobs added to every newly-enabled Ralph project:

```bash
# Pull Request settings
PR_ENABLED=true          # false = revert to old direct-merge behaviour
PR_BASE_BRANCH=""        # empty = auto-detect main branch from git
PR_DRAFT=false           # true = create PRs as GitHub Drafts
```

Same defaults added to `.ralphrc.codex` and `.ralphrc.devin` templates (in `codex/setup_codex.sh` and `devin/setup_devin.sh`).

## Behaviour Matrix

| Scenario | Result |
|---|---|
| Quality gates pass | PR opened, branch preserved as PR head |
| Quality gates fail, Ralph resolves on retry | PR opened on resolved run |
| Quality gates fail, Ralph gives up | PR opened with `quality-gates-failed` label + failure details in description |
| `gh` not installed | Loud startup warning, `PR_CAPABLE=false`, commit + push only |
| No git remote | Loud startup warning, `PR_CAPABLE=false`, commit only |
| `gh` not authenticated | Loud startup warning, `PR_CAPABLE=false`, commit + push only |
| `WORKTREE_ENABLED=false` | Temp branch created, commit, push, PR via `worktree_fallback_branch_pr` |
| `PR_DRAFT=true` | PR created with `--draft` flag |
| `PR_ENABLED=false` | Reverts to old direct-merge behaviour (no PR created) |
